## Why openapi-ng

<CardGrid>
  <Card title="Rust-powered codegen" icon="rocket">
    The engine is a native binary loaded via [NAPI-RS](https://napi.rs).
    The same input always produces identical output.
  </Card>
  <Card title="Angular-first output" icon="puzzle">
    Each operation comes with three flavors — `.observable()`,
    `.resource()`, and `.request()` — matching Angular's current HTTP
    primitives.
  </Card>
  <Card title="Strict OpenAPI subset" icon="approve-check">
    A focused 3.x subset with clear diagnostics. No silent
    misgeneration.
  </Card>
  <Card title="Configurable naming" icon="setting">
    Tune method names and service grouping with template + regex rules,
    via YAML, JSON, or TypeScript config.
  </Card>
  <Card title="Thin, pass-through helpers" icon="open-book">
    Generated methods build the request and forward every
    `HttpClient.request` / `httpResource` option through
    unchanged. The response reaches you untouched.
  </Card>
</CardGrid>

## Install

```bash
# pnpm
pnpm add -D @avsystem/openapi-ng

# npm
npm install -D @avsystem/openapi-ng

# bun
bun add -d @avsystem/openapi-ng

# yarn
yarn add -D @avsystem/openapi-ng
```

Requires Node.js 18+. Pre-built binaries are published for macOS, Linux,
and Windows (x64 and ARM64), with a WASI fallback for other hosts. See
[Runtime & platforms](/reference/runtime/).
